CubeMars Motor Control GUI

CAN-Based Control Interface for High-Torque BLDC Actuators

Executive Summary

Developed a Python-based graphical user interface (GUI) for controlling and monitoring CubeMars AK-series BLDC motors over CAN. The interface supports real-time commanding, telemetry visualization, live parameter tuning, and safety mechanisms for high-torque robotic actuators used in research and prototyping.

Problem & Constraints

Effective testing of high-performance robotic actuators requires a reliable interface for issuing commands, observing state feedback, and adjusting control parameters during operation. The system needed to handle continuous CAN communication, enforce safe operating limits, and remain flexible enough to support rapid iteration without relying on vendor software.

System Overview

The Motor Control GUI is a desktop application written in Python that communicates with CubeMars AK-series motors over a shared CAN bus via a USB-CAN adapter. The software manages motor command generation, feedback decoding, and UI interaction in a single integrated tool suitable for bench testing and actuator characterization.

Control & Telemetry Capabilities

  • MIT Mode Control: Torque, position, and velocity commands supported through CAN messaging.
  • Multi-Motor Operation: Independent addressing and synchronized control of multiple motors on a shared bus.
  • Live Parameter Tuning: Control gains and setpoints adjustable during operation.
  • Real-Time Feedback: Continuous display of motor position, velocity, torque, and temperature.

Implementation Notes

Motor communication is implemented according to the CubeMars CAN protocol specification, with careful handling of message formatting, scaling, and state transitions. The GUI layer is built using Python’s Tkinter framework, enabling a lightweight and responsive interface while maintaining clear separation from low-level communication logic.

Safety & Reliability Features

  • Heartbeat-based motor enable logic
  • Configurable soft torque and position limits
  • Global emergency stop for immediate motor disable
  • Visual indicators for motor state and communication status

Data Handling & Experiment Workflow

Telemetry can be streamed live to the interface and optionally logged for post-test analysis. Parameter presets allow experiments to be repeated under consistent conditions, supporting iterative controller development and hardware testing.

Outcome & Validation

The interface enabled stable real-time control and monitoring of CubeMars AK-series motors during repeated bench and load tests. Live tuning and integrated safety logic improved testing efficiency and reduced the risk of hardware damage during actuator experimentation.

Tools & Skills
