Multi-Motor CAN Control Interface

Real-Time MIT-Mode Control with Telemetry and Safety Interlocks

Executive Summary

Built a Python desktop GUI to command and monitor CubeMars AK-series BLDC actuators over a shared CAN bus. The tool enables real-time setpoint control, live telemetry visualization, on-the-fly gain tuning, and repeatable test workflows, with safeguards designed for high-torque bench experimentation.

Problem & Constraints

Bench testing high-performance actuators requires a fast feedback loop between commanding, observing state, and iterating on parameters. The interface needed to maintain reliable CAN communication, support multiple motors simultaneously, and enforce safe operating limits without relying on vendor software.

System Overview

The application is written in Python and communicates with CubeMars AK-series motors through a USB-CAN adapter. It integrates command generation, feedback decoding, UI controls, and logging into a single tool for actuator characterization, controller iteration, and repeatable experiments.

Control & Telemetry Capabilities

  • MIT Mode Control: Send torque, position, and velocity commands over CAN.
  • Multi-Motor Operation: Independent addressing with synchronized updates on a shared bus.
  • Live Tuning: Adjust gains and setpoints during operation without restarting tests.
  • Real-Time Feedback: Stream position, velocity, torque, and temperature to the UI.

Implementation Notes

Motor messaging follows the CubeMars CAN protocol specification, including scaling, packing/unpacking, and state transitions. The UI is implemented in Tkinter for a lightweight desktop workflow, with low-level CAN/protocol logic kept separate from interface code to improve readability and maintainability.

Safety & Reliability Features

  • Heartbeat enable: Motors remain enabled only while the GUI continues to send periodic keep-alive commands.
  • Soft limits: Configurable torque and position bounds to prevent unsafe commands.
  • Emergency stop: One-click global disable for immediate shutdown.
  • Status indicators: Clear visual state for motor enable, faults, and communication health.

Data Handling & Experiment Workflow

Telemetry can be visualized live and optionally logged for post-test analysis. Parameter presets support repeatable experiments and faster iteration during controller development and hardware debugging.

Outcome & Validation

The GUI supported stable real-time control and monitoring across repeated bench and load tests. Live tuning and integrated safety logic improved iteration speed and reduced the risk of actuator damage during experimentation.
